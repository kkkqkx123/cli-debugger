# Rust 无状态 JDWP CLI 客户端设计方案（命令式、无状态、同步返回）

## 一、必须实现的核心功能（最小可用集）

### 1. 基础连接与协议层（必选）

- JDWP 握手协议实现（Jdwp-Handshake 字符串交换）
- 数据包编解码（命令包/应答包序列化、反序列化）
- 套接字通信（TCP 客户端，单次连接-执行-断开）
- ID 大小获取（objectID/threadID 等字节长度，协议必须）

### 2. 虚拟机基础控制（必选）

- 版本查询（验证连接有效性）
- 虚拟机挂起/恢复
- 全部线程枚举
- 类搜索（按类名/签名查找类 ID）

### 3. 断点核心功能（CLI 核心）

- 行断点设置（类+行号）
- 方法断点设置
- 断点清除（单个/全部）
- 事件监听（断点命中、异常、线程结束）

### 4. 执行控制（命令式操作）

- 继续执行（cont）
- 单步进入（step）
- 单步跳过（next）
- 单步退出（finish）

### 5. 信息获取（同步返回输出）

- 栈帧查看（调用栈）
- 局部变量获取
- 实例/静态字段获取
- 断点命中事件输出
- 线程状态查询

### 6. 无状态核心特性

- **不保存任何会话状态**：每次命令执行都重新建立连接→执行→关闭连接
- **命令原子性**：单个 CLI 命令 = 单次完整 JDWP 交互
- **同步阻塞**：命令执行完成后直接返回结果文本/JSON

## 二、CLI 设计方案（命令式结构）

### 1. CLI 命令格式（直观易用）

```bash
jdwp-cli <COMMAND> [OPTIONS] --host <IP> --port <PORT>
```

### 2. 核心命令清单（对应 JDWP 功能）

| 命令           | 功能         | 参数示例                     |
| -------------- | ------------ | ---------------------------- |
| `version`      | 查询JVM版本  | --host 127.0.0.1 --port 8000 |
| `threads`      | 列出所有线程 | 同上                         |
| `break add`    | 添加行断点   | --class Main --line 10       |
| `break remove` | 删除断点     | --id 1                       |
| `break clear`  | 清除所有断点 | -                            |
| `cont`         | 恢复执行     | -                            |
| `step`         | 单步进入     | -                            |
| `next`         | 单步跳过     | -                            |
| `stack`        | 查看调用栈   | --thread 1                   |
| `locals`       | 查看局部变量 | --thread 1 --frame 0         |
| `classes`      | 搜索类       | --name Main                  |

### 3. 无状态执行流程（固定模板）

```
1. 解析 CLI 参数（命令+目标地址+参数）
2. 创建 TCP 连接 → 完成 JDWP 握手
3. 获取 ID 大小（协议必备）
4. 执行对应 JDWP 命令序列
5. 接收并解析返回数据
6. 关闭连接
7. 格式化输出（纯文本/JSON）
```

---

## 3. 代码架构分层（Rust 实现）

### 第一层：协议层（jdwp_protocol）

- `packet.rs`：包编解码
- `commands.rs`：所有命令枚举+参数结构
- `types.rs`：ID、Value、Location 等类型定义

### 第二层：客户端层（jdwp_client）

- `connection.rs`：连接、握手、发送/接收包
- `vm.rs`：虚拟机操作
- `breakpoint.rs`：断点管理
- `thread.rs`：线程/栈帧/变量操作

### 第三层：CLI 层（bin/jdwp-cli）

- `cli.rs`：命令解析（clap）
- `executor.rs`：命令路由与执行
- `output.rs`：结果格式化输出

## 四、关键技术点处理

### 1. 无状态断点实现

- 每次执行断点命令时**临时注册事件请求**
- 命令执行完成后**自动清理**，不保留状态

### 2. 同步获取输出

- 阻塞等待 JDWP 事件（断点命中/单步完成）
- 捕获事件后立即解析栈、变量并返回
- 超时机制防止命令卡死

### 3. 命令式操作适配

- 不使用 REPL 循环
- 每条命令独立运行
- 输出直接打印到 stdout，可管道/脚本调用

### 4. 错误处理

- 连接失败、命令不支持、类不存在等明确报错
- 返回非 0 退出码，支持脚本判断执行结果

## 五、最简实现步骤（推荐开发顺序）

1. 实现 JDWP 握手 + 版本查询
2. 实现 CLI 框架 + TCP 连接
3. 实现线程枚举 + 类搜索
4. 实现断点添加 + 事件监听
5. 实现执行控制（cont/step/next）
6. 实现栈帧 + 局部变量获取
7. 完善输出格式与错误处理

### 总结

1. **无状态 = 每次命令重连，不存会话**
2. **命令式 = 单条命令=单次操作+直接返回结果**
3. **必须实现**：协议编解码、连接握手、断点、执行控制、栈/变量获取
4. **架构**：三层结构（协议-客户端-CLI），清晰易维护
5. **输出**：同步返回文本/JSON，支持脚本调用

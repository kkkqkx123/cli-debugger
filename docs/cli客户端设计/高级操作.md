# Rust JDWP CLI 客户端 扩展设计方案
## 批量执行 + 配置文件 + 触发器（定时/阈值）
**纯无状态保留、命令式、可脚本化、可配置化**

---

# 一、整体架构扩展（极简、可落地）
```
CLI 命令层 → 配置解析层 → 执行引擎层 → JDWP 协议层 → 目标 JVM
```

新增三大模块：
1. **配置文件系统**（YAML）
2. **操作序列批量执行器**
3. **触发器引擎**（定时 + 阈值）

---

# 二、配置文件格式（YAML，核心）
## 1. 全局连接配置
```yaml
target:
  host: 127.0.0.1
  port: 8000
  timeout: 5s
  retry: 3
```

## 2. 操作序列（批量执行）
```yaml
tasks:
  - name: list_threads
    action: threads

  - name: add_breakpoint
    action: break_add
    class: com.example.Main
    line: 20

  - name: continue_exec
    action: cont

  - name: get_stack
    action: stack
    thread: 1

  - name: get_locals
    action: locals
    thread: 1
    frame: 0
```

## 3. 触发器配置（可选）
```yaml
triggers:
  - name: timer_check
    type: cron
    schedule: "*/5 * * * *"  # 每5秒
    tasks: [list_threads, get_stack]

  - name: thread_threshold
    type: threshold
    metric: thread_count
    operator: gt
    value: 20
    tasks: [add_breakpoint, cont]
```

支持触发器类型：
- `cron`：定时执行
- `threshold`：指标阈值（线程数/堆/类加载数/断点命中数）
- `event`：JDWP 事件触发（断点/异常/类加载）

# 三、批量执行引擎设计
## 核心特性
- **无状态批量执行**
- **顺序执行 / 条件执行**
- **变量传递**（上一步结果传给下一步）
- **失败策略**（continue / abort）
- **输出统一汇总**

## 执行流程
```
1. 加载配置
2. 建立 JDWP 连接（一次连接执行全部任务）
3. 依次执行 task 列表
4. 收集每步结果
5. 关闭连接
6. 输出最终报告
```

## CLI 使用方式
```bash
jdwp-cli exec --config debug.yml
jdwp-cli run task1,task2,task3 --host 127.0.0.1 --port 8000
```

# 四、触发器引擎设计
## 1. 定时触发器（Cron）
- 使用 `cron` 库定时执行任务序列
- 后台守护模式：`jdwp-cli daemon --config monitor.yml`

## 2. 阈值触发器
支持监控指标：
- 线程数量
- 堆使用量
- 类加载数量
- 断点命中次数
- 异常抛出速率
- CPU 使用率（JVM 内部）

触发条件：
`> / < / == / changed`

触发动作：
执行任意 task 序列（断点、dump、stack、cont 等）

## 3. 事件触发器
监听 JDWP 事件：
- 断点命中
- 异常抛出
- 线程启动/结束
- 类加载

自动执行预设操作序列。

# 五、CLI 命令扩展（最终形态）
```
jdwp-cli
  |-- version               # 查询版本
  |-- threads               # 线程列表
  |-- break add/remove/clear# 断点
  |-- cont/step/next/finish # 执行控制
  |-- stack                 # 调用栈
  |-- locals                # 局部变量

  |-- exec --config FILE    # 批量执行配置
  |-- daemon --config FILE  # 后台守护+触发器
  |-- validate --config FILE# 校验配置
```

# 七、关键技术点
## 1. 无状态 + 批量执行兼容
- 一次连接，执行所有任务
- 执行完成断开，不保存任何状态
- 完全满足脚本化、自动化需求

## 2. 操作序列可复用
- 配置文件可版本管理
- 可共享、可重复执行
- 支持多环境切换（dev/test/prod）

## 3. 资源监控轻量高效
- 非侵入式
- 基于 JDWP 原生指标
- 低性能消耗

## 4. 输出格式统一
支持：
- text
- json
- yaml
便于日志采集、监控系统对接。

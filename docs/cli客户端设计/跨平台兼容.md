# Rust JDWP CLI 跨平台兼容：必须特殊处理的功能清单

纯技术要点，无多余信息，直接对应你要开发的**无状态CLI + 批量执行 + 触发器 + JDWP调试**场景。

---

# 一、网络与连接（必须跨平台处理）

1. **TCP 连接超时**
   - Windows / Linux / macOS 超时机制不同
   - Rust 标准库不统一，必须用 `tokio`/`socket2` 统一封装
2. **端口占用检测**
   - Windows 无 `lsof`，macOS/Linux 可用
   - 必须用 Rust 原生 socket 检测，不能调用系统命令
3. **JDWP 共享内存（dt_shmem）**
   - 仅 Windows 支持
   - Linux/macOS 只能用 `dt_socket`
   - 必须做**平台隔离**：Windows 可选，其他平台强制禁用

---

# 二、文件系统与路径（高频坑点）

1. **路径分隔符**
   - Windows `\`
   - Linux/macOS `/`
   - 必须用 `std::path::Path` / `PathBuf`，**禁止硬编码路径**
2. **配置文件加载**
   - Windows 隐藏文件、权限不同
   - 相对路径解析行为不一致
3. **换行符**
   - Windows `\r\n`
   - Unix `\n`
   - 输出日志、批量脚本、YAML 解析必须统一
4. **文件权限**
   - Windows 无 chmod 模型
   - 不要使用 Unix 权限判断逻辑

---

# 三、CLI 命令行与控制台（必须处理）

1. **控制台输出编码**
   - Windows 默认 GBK
   - Linux/macOS UTF-8
   - 输出必须强制 UTF-8，否则乱码
2. **信号处理（Ctrl+C/SIGINT）**
   - Windows 不支持完整 POSIX 信号
   - 必须用 `ctrlc` 跨平台库统一处理
3. **命令行参数解析**
   - Windows 不区分大小写
   - 路径参数带空格解析不同
4. **标准输入输出管道**
   - Windows 管道缓冲机制不同
   - 批量执行输出要做兼容

---

# 四、进程、信号、守护进程（关键）

1. **后台守护进程（daemon）**
   - Linux/macOS：fork 模型
   - Windows：无 fork，必须用 service 或 detached process
   - **你的触发器/定时任务必须重写适配**
2. **子进程管理**
   - Windows `CreateProcess`
   - Unix `fork/exec`
   - 用 Rust `std::process::Command` 可跨平台，但环境变量要处理
3. **进程退出码**
   - Windows 退出码范围不同
   - 统一使用 0=成功，非0=错误

---

# 五、触发器引擎（跨平台重灾区）

1. **定时触发器（Cron）**
   - Linux/macOS 原生支持 cron
   - Windows 无 cron，必须用 Rust 内部定时器（cron-rs + tokio）
2. **系统时间获取**
   - 时区格式不同
   - 统一用 UTC 时间
3. **高分辨率定时器**
   - Windows 精度不同
   - 依赖 `tokio` 时间轮统一

---

# 六、JDWP 相关（必须平台判断）

1. **JVM 启动调试参数**
   - Windows 进程启动参数格式不同
   - 空格、引号处理不同
2. **本地调试地址格式**
   - Windows：`address=8000`
   - Linux/macOS：`address=0.0.0.0:8000`
3. **类路径分隔符**
   - Windows `;`
   - Unix `:`
   - 读取 classpath 必须自动适配

---

# 七、系统监控与阈值触发器（必须跨平台）

1. **CPU 使用率获取**
   - Windows / Linux / macOS 接口完全不同
   - 必须用 `sysinfo` 跨平台库
2. **内存/线程数获取**
   - 不能调用 `ps`/`taskmgr`
   - 用 Rust 原生库
3. **磁盘IO/网络IO**
   - 平台不兼容，统一不实现或用第三方库

---

# 八、Rust 编译与构建（必须处理）

1. **换行符（git autocrlf）**
   - Windows 自动转 \r\n 会破坏配置文件校验
2. **动态链接库**
   - Windows `.dll`
   - Linux `.so`
   - macOS `.dylib`
3. **可执行文件后缀**
   - Windows `.exe`
   - 其他无

---

# 极简总结（你开发时直接对照）

## 必须特殊处理的 8 类功能

1. 网络超时、共享内存（dt_shmem）
2. 文件路径、分隔符、换行符
3. 控制台编码、信号（Ctrl+C）
4. 守护进程（daemon 模式）
5. 定时触发器（Windows 无 cron）
6. 系统监控指标（CPU/内存）
7. JVM 类路径分隔符
8. 进程、退出码、子进程

## 你只要遵守一条规则

**所有系统级操作不调用系统命令（ls/pidof/ps），全部使用 Rust 跨平台库实现**
